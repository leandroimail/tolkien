# Academic Bibliography Manager

Management, validation, and enrichment of the `references.bib` file for an academic article project. Ensures bibliographic integrity as a prerequisite for the Citation↔Bibliography gate.

## When To Use

- Validating completeness of mandatory fields in each BibTeX entry
- Enriching incomplete entries via the OpenAlex API
- Resolving DOI → full metadata → BibTeX
- Detecting and removing duplicates (by DOI or title)
- Checking if articles have been retracted (`is_retracted`)
- Formatting `.bib` according to PRD style (APA, IEEE, ABNT, etc.)
- Before the Citation↔Bibliography gate (prerequisite)

## When Not To Use

- To manage **in-text** citations in the draft → use `academic-citation-manager`
- To search for literature and screening → use `academic-researcher`
- To format the final paper in LaTeX → use the `latex` skill

## Prerequisites

1. **`research/references.bib`** — raw BibTeX file (generated by the researcher or manually)
2. **`prd.md`** — to identify citation style and discipline field
3. Internet connection (for OpenAlex API — optional but recommended)

## Method

### Phase 1: Loading and Inventory

1. Read `research/references.bib` and parse all entries
2. Classify each entry by type (`@article`, `@inproceedings`, `@book`, `@misc`)
3. Generate inventory: total entries, types, present/missing fields

### Phase 2: Mandatory Fields Validation

Mandatory fields by type:

| Type | Mandatory Fields |
|------|-------------------|
| `@article` | author, title, journal, year, volume, pages |
| `@inproceedings` | author, title, booktitle, year |
| `@book` | author/editor, title, publisher, year |
| `@misc` | author, title, year, url, note |

Run validator:
```bash
python scripts/validate_bib.py research/references.bib
```

### Phase 3: Duplicate Detection

Duplicate criteria:
- Identical DOI between two entries
- Title similarity ≥ 90% (normalized lowercase, no punctuation)
- Same author + year + similar title

```bash
python scripts/check_bib_duplicates.py research/references.bib
```

### Phase 4: Enrichment via OpenAlex

For entries with missing fields that have a DOI:

```bash
# Polite pool: include email for 10 req/s
curl -s "https://api.openalex.org/works/https://doi.org/{DOI}?mailto={email}"
```

OpenAlex → BibTeX mapping:
- `title` → `title`
- `authorships[*].author.display_name` → `author` ("Last, First and ..." format)
- `primary_location.source.display_name` → `journal`
- `publication_year` → `year`
- `biblio.volume` → `volume`
- `biblio.first_page`-`biblio.last_page` → `pages`
- `doi` → `doi`
- `is_retracted` → alert flag

### Phase 5: Retraction Check

For each entry with a DOI, check `is_retracted` via OpenAlex:
- If `true`: issue a **CRITICAL ALERT** and suggest removal or replacement

### Phase 6: Formatting and Output

1. Standardize `.bib` formatting:
   - Consistent field order
   - Citation keys in `FirstAuthorYearKeyword` format
   - Special characters protected with `{}`
   - Pages with `--` (double dash)
   - **ABNT Specific (if style = ABNT/ABNT2)**:
     - Ensure electronic sources contain `url` and `urlaccessdate` fields.
     - Ensure authors are properly separated by `and` for correct capitalization by `abntex2`.
2. Write updated `research/references.bib`
3. Generate report in `review/bibliography-report.md`

## Self-Review

### Deterministic
- [ ] 0 entries with missing mandatory fields
- [ ] 0 duplicates detected
- [ ] 0 untreated retractions
- [ ] All citation keys are unique

### Agentic
- Evaluate coverage: do the references adequately cover the PRD questions?
- Check recency: sources from the last 5-10 years for active fields?
- Recommend additional sources if thematic coverage is insufficient

## Output

```markdown
### Bibliography Report
- **Total entries**: N
- **Validated**: N/N (100%)
- **Enriched via OpenAlex**: N entries
- **Duplicates removed**: N
- **Retractions found**: N (CRITICAL if > 0)
- **Status**: ✅ Ready for citation gate | ❌ N issues to resolve
```
