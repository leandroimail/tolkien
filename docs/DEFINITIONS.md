# Tolkien — Definitions & Reference

Complete glossary, agent inventory, skill catalog, and directory specification for the Academic Article Production Multi-Agent System.

---

## Glossary

| Term | Definition |
|------|-----------|
| **Academic SDD** | Spec-Driven Development adapted for scientific writing. Enforces strict sequencing: PRD → Plan → Execution. No phase may start before the previous one is validated. |
| **Agent** | A high-level coordinator that orchestrates multiple skills to achieve a complex, multi-step goal (e.g., running the full literature research pipeline). Agents are invoked by trigger phrases in the Claude Code or OpenCode prompt. |
| **Skill** | An atomic, single-purpose capability that performs one specific task (e.g., searching OpenAlex, compiling LaTeX). Skills are invoked via `/skill-name` commands. |
| **Gate** | A mandatory human checkpoint that blocks pipeline progression until defined quality criteria are met. There are 6 gates (G1–G5.5). |
| **PRD** | Product Requirements Document — in this context, the academic paper specification. Defines research questions, target venue, citation style, language, and constraints. Stored as `prd.md`. |
| **prd.md** | The paper requirements file generated by `academic-prd`. It is the single source of truth for all pipeline decisions. |
| **plan.md** | The implementation roadmap generated by `academic-plan` from `prd.md`. Lists all 9+ phases with tasks, deliverables, and acceptance criteria. |
| **references.bib** | The BibTeX bibliography file. Managed by `academic-bibliography-manager`. Used as the sole source of citations by `academic-citation-manager`. |
| **IMRaD** | Introduction, Methods, Results, and Discussion — the standard structure for empirical research papers. |
| **5-D Review** | The five-dimension peer review simulation run by `academic-reviewer`: (1) Editorial Fit, (2) Technical Soundness, (3) Methodology Rigor, (4) Interpretation Quality, (5) Devil's Advocate critique. |
| **process-record.md** | The human-AI collaboration log generated in Phase 9. Documents every decision, gate outcome, and revision during the pipeline run. |
| **OpenAlex** | Free, open scholarly database used by `academic-researcher` and `academic-bibliography-manager` to search literature and enrich BibTeX entries. No API key required. |
| **Citation↔Bibliography Gate** | The G4 validation that checks: (1) every `\cite{key}` in the draft has a matching entry in `references.bib`; (2) every entry in `references.bib` is cited in the draft; (3) every `.bib` entry has all mandatory fields. |
| **Slug** | A short, URL-safe identifier for a paper project (e.g., `vector-retrieval`). Used as the directory name: `paper-{slug}/`. |
| **TinyTeX** | A lightweight LaTeX distribution installed by `install_skills_deps.sh` when a full TeX distribution is not found. Provides `pdflatex`, `bibtex`, and `tlmgr`. |
| **.venv** | The project-local Python virtual environment created at the repository root by `install_skills_deps.sh`. Must be activated before running any Python-based skill. |

## Agents

tolkien includes 6 specialized agents. All agent definitions are stored under `.claude/agents/` (Claude Code) and `.agents/agents/` (OpenCode).

### academic-orchestrator

| Property | Value |
|----------|-------|
| **Purpose** | Master coordinator. Executes the full 10-phase pipeline, enforces gates, and manages project state. |
| **Triggers** | `/academic-orchestrator`, `"start academic pipeline"`, `"write full article"`, `"academic pipeline"`, `/status` |
| **Dispatches** | All pipeline agents and skills, in order |
| **File** | `.claude/agents/academic-orchestrator.md` |

### research-agent

| Property | Value |
|----------|-------|
| **Purpose** | Systematic literature search, source triage, and bibliography synthesis. |
| **Triggers** | `/research-agent`, `"research for paper"`, `"search literature and validate bib"` |
| **Dispatches** | `academic-researcher`, `academic-bibliography-manager`, `web-browser-search-agent` |
| **File** | `.claude/agents/research-agent.md` |

### writing-agent

| Property | Value |
|----------|-------|
| **Purpose** | Full-text section drafting, scientific media generation, and register humanization. |
| **Triggers** | `/writing-agent`, `"draft full article"`, `"write and humanize"` |
| **Dispatches** | `academic-writer`, `academic-media`, `academic-humanizer` |
| **File** | `.claude/agents/writing-agent.md` |

### review-agent

| Property | Value |
|----------|-------|
| **Purpose** | Citation↔Bibliography validation and 5-dimension peer review with re-review cycles. |
| **Triggers** | `/review-agent`, `"review full article"`, `"execute academic review"` |
| **Dispatches** | `academic-citation-manager`, `academic-bibliography-manager`, `academic-reviewer`, `web-browser-search-agent` |
| **File** | `.claude/agents/review-agent.md` |

### paper-generator-agent

| Property | Value |
|----------|-------|
| **Purpose** | Converts the reviewed draft into a finalized PDF or DOCX using LaTeX or Word. |
| **Triggers** | `/paper-generator`, `"generate final paper"`, `"compile LaTeX"`, `"generate paper PDF"`, `"export paper"` |
| **Dispatches** | `latex`, `latex-template-converter`, `pdf`, `docx` |
| **File** | `.claude/agents/paper-generator-agent.md` |

### web-browser-search-agent

| Property | Value |
|----------|-------|
| **Purpose** | Web search and browser automation for grey literature, full-text retrieval, and retraction checks. |
| **Triggers** | Invoked internally by `research-agent` and `review-agent`; also: `/web-browser-search-agent`, `"search the web"`, `"browse URL"`, `"validate DOI online"`, `"check URL"`, `"open website"`, `"extract web content"` |
| **Dispatches** | `web-browser-search`, `duckducksearch`, `agent-browser`, `playwright-cli` |
| **File** | `.claude/agents/web-browser-search-agent.md` |

## Skills Inventory

tolkien includes 21 skills (9 pipeline skills + 12 tool skills) across two categories. All skill definitions live under `.claude/skills/` (Claude Code) and `.agents/skills/` (OpenCode).

### Pipeline Skills

These skills implement the academic writing workflow.

| Skill | Trigger | Purpose | Pipeline Phase |
|-------|---------|---------|---------------|
| `academic-prd` | `/academic-prd` | Structured interview to generate `prd.md` | Phase 0 |
| `academic-plan` | `/academic-plan` | Generates `plan.md` from an approved PRD | Phase 1 |
| `academic-researcher` | `/academic-researcher` | Systematic literature search via OpenAlex API; produces `literature.md` and `references.bib` | Phase 2 |
| `academic-writer` | `/academic-writer`, `"write section"` | Drafts paper sections in IMRaD or thematic format with field-specific register | Phase 3–4 |
| `academic-citation-manager` | `/academic-citation-manager`, `"verify citations"` | Validates all `\cite{key}` references against `references.bib` | Phase 5 |
| `academic-bibliography-manager` | `/academic-bibliography-manager`, `"validate bibliography"` | Validates, deduplicates, and enriches `references.bib` via OpenAlex | Phase 5 |
| `academic-humanizer` | `/academic-humanizer`, `"humanize"` | Adjusts register and removes AI-writing markers while preserving academic rigor. Supports EN, pt-BR, FI. | Phase 6 |
| `academic-reviewer` | `/academic-reviewer`, `"review article"` | Simulates a 5-D reviewer panel: EIC + 3 domain reviewers + Devil's Advocate | Phase 7 |
| `academic-media` | `/academic-media`, `"create figure"` | Generates publication-quality figures, schematics, and exploratory data analysis (EDA) plots | Phase 4, 8 |

### Tool Skills

These skills handle document formats and external system integration.

| Skill | Trigger | Purpose |
|-------|---------|---------|
| `latex` | `/latex` | Full LaTeX compilation, formatting, and error resolution |
| `latex-template-converter` | `/latex-template-converter` | Adapts documents to conference-specific LaTeX templates (ACM, IEEE, NeurIPS, SBBD, etc.) |
| `pdf` | `/pdf` | PDF text/table extraction, merging, splitting, watermarking, and creation |
| `docx` | `/docx` | Word document creation, editing, and formatting |
| `xlsx` | `/xlsx` | Spreadsheet reading, editing, and formula computation |
| `agent-browser` | `/agent-browser` | Browser automation: navigate, fill forms, click, screenshot, extract data |
| `playwright-cli` | `/playwright-cli` | Playwright-based browser testing and automation |
| `web-search` | `/web-search` | Web search returning ranked results with snippets and URLs |
| `web-browser-search` | `/web-browser-search` | Unified DuckDuckGo/Brave search + browser navigation |
| `duckducksearch` | `/duckducksearch` | DuckDuckGo search with image, video, and news filters |
| `creating-skills` | `/creating-skills` | Framework for developing, validating, and deploying new skills |
| `multi-ide-artifacts` | `/multi-ide-artifacts` | Converts agent artifacts across Claude Code, OpenCode, Codex, and other IDEs |

## Project Directory Structure

Each paper project must follow this structure exactly. The `output/` directory must contain all final deliverables.

```text
{root}/paper-{slug}/
├── prd.md                     # Paper requirements (generated by academic-prd)
├── plan.md                    # Implementation roadmap (generated by academic-plan)
├── process-record.md          # AI-human collaboration log (generated in Phase 9)
│
├── research/
│   ├── literature.md          # Annotated sources + synthesis
│   ├── search-strategy.md     # Documented search methodology (databases, queries)
│   └── references.bib         # BibTeX bibliography — source of truth for citations
│
├── draft/
│   ├── outline.md             # Approved structure + word count allocation
│   ├── abstract.md
│   ├── introduction.md
│   ├── methodology.md
│   ├── results.md
│   ├── discussion.md
│   └── conclusion.md
│
├── review/
│   ├── review-report.md       # 5-D peer review results + scores
│   ├── citation-report.md     # Citation↔Bibliography validation results
│   └── revision-log.md        # Revision history with rationale
│
└── output/                    # ALL final deliverables go here
    ├── paper.tex
    ├── paper.pdf
    ├── paper.docx
    └── figures/
```

**Valid root directories:**

```
projects/      papers/      .projects/      .papers/
```

Example full path: `/path/to/tolkien/papers/paper-vector-retrieval/`

## Configuration Directories

tolkien maintains two parallel configuration trees for cross-IDE compatibility:

| Directory | IDE | Contents |
|-----------|-----|---------|
| `.claude/` | Claude Code (CLI) | `agents/`, `skills/`, `settings.local.json` |
| `.agents/` | OpenCode | `agents/`, `skills/` |

Both directories contain identical agent and skill definitions. When updating a skill or agent, update both directories or use the `multi-ide-artifacts` skill to synchronize them.

### Settings

`.claude/settings.local.json` controls harness permissions for the Claude Code runner. It defines which Bash commands agents are allowed to execute (LaTeX compilation, file operations, git commands, Python scripts).

### Python Environment

The `.venv/` directory at the repository root is the shared Python environment for all Python-based skills. It must be activated before invoking any skill that uses Python directly:

```bash
# Activate
source .venv/bin/activate

# Deactivate when done
deactivate
```
